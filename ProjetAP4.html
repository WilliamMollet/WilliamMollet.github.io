Context

La MAISON DES LIGUES DE LA LORAINNE, établissement du Conseil Régional de Lorraine,
est responsable de la gestion du service des sports et en particulier des ligues sportives ainsi que d’autres structures hébergées.
M2L souhaiterais développer une application mobile pour la gestion des produits du site e-commerce.

La solution proposée est le développement d’une application en Flutter.

Ce qu'est Flutter

Flutter est un framework open-source développé par Google qui permet de créer des applications multiplateformes pour mobiles, web et ordinateurs à partir d'un code source unique. C'est un framework puissant et flexible offrant une productivité élevée et des performances graphiques de qualité. Il est principalement utilisé pour développer des applications mobiles pour Android et iOS.

L'application

Pour répondre aux attentes envers cette application, 5 pages on été mises en places :
  • Une page de connexion. Celle ci s'occupe de verifier que l'utilisateur de l'application possède bien un compte sur notre base de données ainsi que ce compte soit bien un compte administrateur.
  • Une page de gestion de produit. Sur cette page, il y a une liste des produits présents dans la base de données. En cliquant sur l'un des produits un menu déroulant s'ouvre, donnant accès a des champs de textes permettant de changer les données d'un produit. Il y a aussi 2 boutons, l'un permettant la sauvegarde, et donc l'envoi en base de données, des changements dans les champs de textes, l'autre sert à la suppression d'un produit.
  • Une page d'ajout de produit, disponible depuis un bouton flottant sur la page de gestion. Cette page contient un formulaire a remplir pour l'ajout d'un produit ainsi que le bouton d'envoi.
  • Une page de gestion d'utilisateur. Elle fait la même chose que pour la page gestion de produit mais pour les utilisateurs
  • Une page d'ajout d'utilisateur. Elle fait la même chose que pour la page d'ajout de produit mais pour les utilisateurs

L'API et Base de données

Dans le cadre de ce projet, et du projet de site Web, une base de données et une API ont dû être créer. L'API, en nodeJS, permet la connexion du site et de l'application a la base de données et la base de données, faite sur MariaDB, stocke les données utilisés par le site et l'application.
